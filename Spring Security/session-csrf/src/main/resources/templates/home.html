세션 관리 정책 & CSRF 보호 테스트

IF_REQUIRED (기본값)

필요할 때만 세션을 생성합니다. 일반적인 웹 애플리케이션에서 사용하는 기본 정책입니다.
로그인 테스트 대시보드 (인증 필요)

ALWAYS

모든 요청에 대해 세션을 생성합니다. 메모리 사용량이 높지만 완전한 상태 추적이 가능합니다.
로그인 테스트 대시보드 (인증 필요)

NEVER

Spring Security가 세션을 생성하지 않습니다. 기존 세션이 있다면 활용합니다.
로그인 테스트 대시보드 (인증 필요)

STATELESS (REST API)

세션을 전혀 사용하지 않습니다. JWT 등 토큰 기반 인증이나 REST API에 적합합니다.
공개 API 테스트

CSRF 보호 테스트

Cross-Site Request Forgery 공격을 방어하는 CSRF 토큰의 동작을 확인합니다.
